# xAI 额度独立卡片页设计

## 背景

new-api 当前在“认证文件”表格中展示 xAI 的 SuperGrok 套餐、月度账单限额、剩余金额、按需上限和账期结束时间。该表格适合绑定管理，但无法清晰承载 xAI 周限额、产品用量、按量付费和月度额度等多组进度信息。

本次新增独立的“xAI 额度”页面，参考 Cli-Proxy-API-Management-Center 的卡片信息结构，并使用 new-api 现有组件、主题变量和交互规范重新实现。

## 目标体验

用户进入“xAI 额度”后，应立即看到自己有多少个 xAI 认证文件、每个账户所属套餐、各额度窗口的已用/剩余情况和重置时间，并能单独刷新任一账户。管理员与普通用户看到的账户范围继续遵循现有 Cliproxy 绑定权限。

认证文件的创建、编辑、启停和删除仍在“认证文件”页面完成。额度页只负责查看与刷新，避免把账户管理和额度监控混在同一页面。

## 范围

### 包含

- 新增 `/xai-quota` 路由和侧边栏“xAI 额度”入口。
- 展示当前角色可访问的 xAI 认证文件数量。
- 使用响应式卡片展示套餐、周限额、产品用量、按量付费、月度额度和重置时间。
- 支持单卡刷新额度。
- 支持完整账号名与脱敏账号名切换，并在本地保存偏好。
- xAI 刷新时并行请求周度和月度账单接口。
- 支持双接口部分成功：保留并展示可用数据，同时提示另一部分刷新失败。
- 保留认证文件表格中现有的 xAI 摘要展示。
- 所有新增用户文案支持现有六种前端语言的 i18n 文件结构。

### 不包含

- 不把 GLM、DeepSeek 和 xAI 合并为统一额度中心。
- 不在 xAI 额度页提供创建、编辑、启停或删除认证文件的入口。
- 不调整 xAI 实际模型调用、鉴权或计费逻辑。
- 不新增后台定时刷新任务。
- 不修改 Classic 前端；当前仓库只存在 Default 前端实现。

## 方案选择

### 方案一：扩展认证文件表格

直接把周限额、产品用量和月度额度继续塞入现有“用量”单元格。改动较小，但表格会过于拥挤，移动端和多产品用量的可读性差，也会继续混合管理与监控职责。

### 方案二：独立 xAI 额度页（采用）

保留认证文件表格作为管理入口，新增专门的卡片页展示额度，并复用现有绑定列表和单项刷新权限。这能完整承载参考图的信息层级，改动边界清晰，后续也能独立优化额度展示。

### 方案三：统一额度中心

将 xAI、GLM 和 DeepSeek 合并为同一套 provider-driven 页面。这是更高杠杆的长期方向，但会扩大到既有额度页面重构、统一数据契约和迁移，超出本次适配范围。

│ ┌─────────────────────┐  ┌─────────────────────┐             │
│ │ [xAI] account.json  │  │ [xAI] account.json  │             │
│ │ ------------------- │  │ ------------------- │             │
│ │ 套餐  SuperGrok     │  │ 套餐  SuperGrok     │             │

│ │ [剩余额度进度条]    │  │ [剩余额度进度条]    │             │
│ │ API 使用   已用 45% │  │ API 使用  已用 100% │             │
│ │ [剩余额度进度条]    │  │ [剩余额度进度条]    │             │
│ │ 按量付费  未启用     │  │ 按量付费  未启用     │             │
│ │ 月度额度 88% ...     │  │ 月度额度 73% ...     │             │
│ │ [剩余额度进度条]    │  │ [剩余额度进度条]    │             │
│ │          [刷新额度] │  │          [刷新额度] │             │
│ └─────────────────────┘  └─────────────────────┘             │

### 响应式规则

- 小于 `md`：单列卡片。
- `md` 至 `2xl`：两列卡片。
- `2xl` 及以上：三列卡片。
- 卡片内部元信息在空间不足时允许换行，文件名使用省略号并通过 `title` 提供完整值。
- 操作按钮保持至少 40px 的可点击高度，并提供可见键盘焦点。

## 卡片信息设计

### 卡片头部

- xAI 类型 Badge。
- 认证文件名。
- 完整账号模式显示原始文件名；脱敏模式隐藏邮箱主体中间部分。

### 套餐

- 月度限额为 `$150.00` 时显示 `SuperGrok`。
- 月度限额为 `$1,500.00` 时显示 `SuperGrok Heavy`。
- 其他已返回套餐优先使用后端套餐值；无法识别时显示 `-`。

### 周限额

- 文案显示“已用 N%”。
- 同行显示周周期起止时间和重置时间。
- 进度条宽度表示剩余比例 `100 - 已用比例`，与参考页面一致。

### 产品用量

- 为周度接口返回的每个 `productUsage` 项生成一行。
- 产品名 `Api` 翻译为“API 使用”；未知产品保留服务端名称。
- 文案显示“已用 N%”，进度条显示剩余比例。

### 按量付费

- `onDemandCap <= 0` 时显示“未启用”，不显示进度条。
- 已启用时显示剩余百分比、剩余金额/上限金额，并显示剩余比例进度条。
- 按量已用金额优先使用服务端 `onDemandUsed`；缺失时使用 `max(used - monthlyLimit, 0)` 推导。

### 月度额度

- 显示剩余百分比。
- 显示“剩余金额 / 月度总额”。
- 显示月度账期结束时间。
- 进度条显示剩余比例。

### 进度颜色

进度条统一表达“剩余额度”：

- 剩余 `>= 70%`：绿色。
- 剩余 `30%–69%`：橙色。
- 剩余 `< 30%`：红色。

## 后端数据流

### 上游请求

xAI 认证文件刷新时并行发起：

1. `GET https://cli-chat-proxy.grok.com/v1/billing?format=credits`
   - 周度使用比例。
   - 周期开始和结束时间。
   - 产品用量列表。
2. `GET https://cli-chat-proxy.grok.com/v1/billing`
   - 月度账单限额与已用金额。
   - 按量付费上限和已用金额。
   - 月度账期开始和结束时间。

请求沿用现有 Cliproxy API Call 代理和认证文件权限，不在 new-api 前端直接访问 xAI。

### 归一化快照

现有字段继续保存月度数据：

- `last_usage_tokens`：月度总已用金额，单位为美分。
- `last_usage_quota`：月度包含额度，单位为美分。
- `last_plan_type`：归一化后的套餐名。
- `last_xai_on_demand_cap`：按量付费上限，单位为美分。
- `last_xai_billing_period_end_at`：月度账期结束时间。

新增字段：

- `last_xai_weekly_percent`：周额度已用百分比。
- `last_xai_weekly_period_start_at`：周周期开始时间。
- `last_xai_weekly_period_end_at`：周周期结束/重置时间。
- `last_xai_product_usage`：产品用量数组的 JSON 文本。
- `last_xai_on_demand_used`：按量付费已用金额，单位为美分。

产品用量以 `TEXT` 保存，保证 SQLite、MySQL 和 PostgreSQL 兼容。序列化和反序列化必须使用项目 `common` JSON 包装函数。

### 刷新合并规则

- 周度与月度均成功：覆盖完整 xAI 快照并清空 `last_error`。
- 仅周度成功：更新周度和产品数据，保留已有月度数据，并记录月度刷新警告。
- 仅月度成功：更新月度和按量数据，保留已有周度数据，并记录周度刷新警告。
- 两者均失败：不覆盖已有快照，保存错误信息并返回刷新失败。
- 任一成功时更新 `last_refreshed_at`。

## API 与权限

- 继续使用 `POST /api/cliproxy/auth-files/bindings/:id/refresh-usage` 刷新单个账户。
- 绑定列表接口增加可选的 `type=xai` 过滤条件，用于服务端筛选 xAI 绑定并返回准确总数。
- xAI 类型匹配兼容 `xai-*`、`xai_*` 文件名，以及 `xai`、`SuperGrok`、`SuperGrok Heavy` 套餐值。
- 管理员可以看到所有匹配的 xAI 绑定；普通用户只能看到绑定给自己的账户。
- 页面不暴露认证文件凭据、代理配置或上游 Token。

## 前端结构

新增：

- `web/default/src/routes/_authenticated/xai-quota/index.tsx`
- `web/default/src/features/xai-quota/index.tsx`
- `web/default/src/features/xai-quota/lib/xai-usage.ts`
- `web/default/src/features/xai-quota/lib/xai-usage.test.ts`

复用：

- `cliproxy-auth-files/api.ts` 中的绑定列表与单项刷新请求。
- `cliproxy-auth-files/types.ts` 中的绑定类型。
- `SectionPageLayout`、`Card`、`Badge`、`Button`、`Progress`、`Alert` 和现有格式化函数。

不移动或重构现有认证文件页面；只为共享 API 增加 `type` 查询参数和新增字段类型。

## 页面状态与错误处理

- 首次加载：显示与卡片尺寸接近的 skeleton，避免布局跳动。
- 空状态：说明尚未绑定 xAI 认证文件，并提供前往“认证文件”页面的链接；普通用户只显示说明，不显示创建操作。
- 单卡刷新：只禁用当前卡片按钮并显示旋转图标，不阻塞其他账户。
- 部分成功：继续展示最新可用数据，在卡片内显示非阻断警告。
- 全部失败：保留旧快照，在卡片内显示错误信息并允许重试。
- 列表失败：使用页面级错误提示和重试按钮。

## 国际化与账号隐私

- 所有新增文案通过 `useTranslation()` 和英文源键调用。
- 同步更新 `en`、`zh`、`fr`、`ru`、`ja`、`vi` locale 文件。
- 账号显示偏好保存在 localStorage，不上传服务端。
- 默认采用脱敏模式；切换按钮包含图标、可见文字和 `aria-pressed`。

## 风险与约束

- xAI 上游两个账单响应可能独立变化，因此解析必须兼容 camelCase 与 snake_case，并允许单侧缺失。
- `productUsage` 是开放列表，前端不能假定只存在 `Api` 一项。
- 旧数据库记录没有新增字段时按零值/空数组展示，不需要数据回填任务。
- 本次保持现有刷新入口和权限模型，不引入新的凭据存储或后台任务。

## 设计自检

- 10x 检查：统一额度中心是长期更高杠杆方向，但本次会导致 GLM/DeepSeek 页面和数据契约一起重构；独立 xAI 页以较小额外成本保留未来抽象空间，同时直接解决当前用户体验。
- Platonic Ideal：额度页的单一任务是让用户在几秒内判断“哪个账户快没额度、什么时候恢复、能否继续付费使用”，因此卡片只保留判断和刷新所需信息，管理操作留在认证文件页。
- 规格无待定项，页面、数据、权限、错误状态和验证边界一致。
